Simple Saver

Simple Saver is a responsive web application that gives people the ability to easily record and view data on their income and expenses. Simple Saver displays data on users' finances so that they can easily adapt their spending habits to reach their financial goals.

My Process

User Flow

This user flow was created to address the needs highlighted in the following user stories:

1. "I want to see a dashboard of my finances clearly and visually, so that I can see how much I am spending on what at a glance."

2. "I want to be able to view, filter, and interact with information on my finances in a variety of ways, so that I can access further details when I want to."

3. "I need to be able to create a profile, so that the financial advice I receive is personalized to me, my spending habits, and tailored to my goals."

Low and Mid Fidelity Wireframes

A set of low fidelity wireframes were created to address one of the user flows:

"I want to see a dashboard of my finances clearly and visually, so that I can see how much I am spending on what at a glance."

These screens allow users to login or create an account, see an overview of their finances, go into details of their spending, and view spending category breakdowns.

After my low-fidelity wireframes were created, a greyscale design was made using Adobe XD. A screen was added to give the option of either logging in or creating an account.

Brand Guidelines

Brand guidelines were created to develop a unified design and to help convey the guiding principles of reliability, simplicity, and effectiveness.

User Testing

At this stage, I sent 4 participants a scenario-based prototype test using the mid-fidelity wireframes. The following feedback was given:

"Love the visual graphs. Some color would be nice. I was hoping to see an actual list of specific expenditures for the month when I clicked on the bar for December."

"Categories is always good to have, color the bars to the categories."

"I didn't initially know where to go for detailed statistics of finances since when I think finances I think net worth and not so much my spending."

Changes made included adding more colors for differentiation, adding more metrics, and making some items smaller.

Final Product

Conclusion

I chose this project because I wanted to design an app which I believe would be a bit more challenging to design since finance isn't traditionally thought of as a very visually appealing subject. However, I really enjoyed creating a clean and sophisticated product. I ran into issues with user testing in greyscale since many users didn't realize that was intentional and most of the comments I received were about adding more color. So next time, this is something I would make clear beforehand.
